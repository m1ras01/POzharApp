## Backend (Node.js + Express + Prisma)

| Файл | Назначение |
|------|------------|
| `backend/package.json` | Зависимости и скрипты (npm run dev, db:migrate и т.д.) |
| `backend/tsconfig.json` | Настройки TypeScript |
| `backend/env.example.txt` | Пример .env (скопировать в .env и заполнить) |
| `backend/prisma/schema.prisma` | Схема БД: User, Notification, Message, ActionLog, DeliveryLog |
| `backend/prisma/seed.ts` | Создание первого администратора (логин/пароль из .env или сгенерированные) |
| `backend/prisma/migrations/.../migration.sql` | SQL миграция для PostgreSQL |
| `backend/src/index.ts` | Точка входа: Express, CORS, подключение маршрутов |
| `backend/src/lib/prisma.ts` | Клиент Prisma (подключение к БД) |
| `backend/src/lib/logAction.ts` | Запись действий в ActionLog |
| `backend/src/middleware/auth.ts` | JWT-авторизация и проверка ролей |
| `backend/src/routes/auth.ts` | POST /api/auth/login — вход |
| `backend/src/routes/notifications.ts` | CRUD уведомлений, фильтры, статистика |
| `backend/src/routes/messages.ts` | Отправка и список сообщений |
| `backend/src/routes/users.ts` | Список пользователей, создание, редактирование, удаление |
| `backend/src/routes/reports.ts` | Отчёты за период |
| `backend/src/routes/settings.ts` | Настройки уведомлений, запрос кода Telegram |
| `backend/src/routes/logs.ts` | Просмотр логов действий (для админа) |

## Как открыть проект в Cursor

1. **File → Open Folder** (или Ctrl+K Ctrl+O).
2. Выберите папку **POzharApp** (та, где лежат `backend`, `frontend`, `README.md`).
3. В левой панели (Explorer) появятся все папки и файлы из таблиц выше.

Если открыта другая папка — вы не увидите код FireNotify. Нужно открыть именно **POzharApp**.
