# FireNotify System

Система мониторинга и обработки пожарных уведомлений.

## Цель проекта

Программная система для приёма, обработки и передачи пожарных уведомлений администраторам, отслеживания статуса сообщений и действий сотрудников.

## Роли пользователей

| Роль | Возможности |
|------|-------------|
| **Администратор** | Все уведомления, получение сообщений, назначение ответственных, управление пользователями |
| **Оператор** | Приём уведомлений, проверка данных, отправка сообщений администратору |
| **Наблюдатель** | Только просмотр уведомлений |

## Структура проекта

- `frontend/` — React-приложение (интерфейс)
- `backend/` — Node.js API (бизнес-логика, БД)
- `docs/` — ТЗ и документация

## Технологии

- **Frontend:** React, TypeScript, Vite
- **Backend:** Node.js, Express, TypeScript
- **БД:** PostgreSQL
- **Уведомления:** Push, SMS, Email, Telegram (интеграции)

## Запуск

### Требования

- Node.js 18+
- PostgreSQL 14+
- npm или yarn

### Backend

```bash
cd backend
cp .env.example .env   # настроить DATABASE_URL и др.
npm install
npm run db:migrate    # применить миграции
npm run dev
```

API: http://localhost:3001

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Приложение: **http://localhost:5173** — откройте этот адрес в браузере (Chrome, Edge, Firefox).

**Если интерфейс не отображается:** убедитесь, что в папке `frontend` выполнены `npm install` и `npm run dev`, затем откройте в браузере http://localhost:5173. Бэкенд должен быть запущен на порту 3001 (или настройте proxy в `frontend/vite.config.ts`).

## Основные страницы

1. **Вход** — логин, пароль
2. **Dashboard** — счётчики, последние события, быстрые действия
3. **Уведомления** — список, фильтры, поиск
4. **Детали уведомления** — источник, адрес, статус, комментарии, «Отправить администратору»
5. **Отправка сообщения** — получатель, текст, вложения
6. **Пользователи** — список, создание, редактирование, удаление
7. **Отчёты** — период, генерация, экспорт PDF/Excel
8. **Настройки** — уведомления, роли, общие параметры
9. **Центр уведомлений** — Telegram, Push/SMS/Email, история доставки

## Статусы уведомлений

Новое → В обработке → Проверено / Ложное → Закрыто

## Безопасность

- HTTPS, хеширование паролей (bcrypt), разграничение прав (RBAC).
