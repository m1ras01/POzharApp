# Тестирование изменений по тендерной документации

Перед тестами один раз выполните (в корне проекта или в указанных папках):

```cmd
cd backend
npx prisma generate
npx prisma db push
npm install
cd ..
```

Запуск приложения: двойной клик по **Запустить всё.bat** или в терминале из корня: `npm run dev`.

---

## 1. Зоны ответственности

| Действие | Где | Ожидание |
|----------|-----|----------|
| Открыть заявку | Уведомления → любая заявка → Подробнее | В карточке есть блок «Ответственный (зона ответственности)» |
| Войти как **Администратор** | Логин admin / ваш пароль | В форме редактирования заявки есть выпадающий список «Ответственный (назначение по зоне ответственности)» |
| Выбрать ответственного и нажать «Сохранить» | Карточка заявки | Заявка сохраняется, в списке у этой заявки в колонке «Ответственный» отображается выбранный пользователь |
| Фильтр по подразделению | Уведомления → «Все подразделения» → выбрать, напр. КОТЭС | Список заявок фильтруется по выбранному подразделению |

---

## 2. Расширенный поиск

| Действие | Где | Ожидание |
|----------|-----|----------|
| Фильтр «Все авторы» | Уведомления | Выпадающий список авторов (кто создавал заявки); при выборе — фильтрация по автору |
| Фильтр «Все типы проблемы» | Уведомления | Список типов (Пожарный сигнал, АПС, Ложное срабатывание и т.д.); при выборе — фильтрация |
| Поле «Местонахождение объекта» | Уведомления | Ввод части адреса → список заявок фильтруется по адресу |
| Комбинация фильтров | Выбрать статус + подразделение + автор | Работают вместе (комбинированный поиск) |
| Таблица заявок | Уведомления | Есть колонки «Автор» и «Тип проблемы» |

---

## 3. Редактирование заявки и комментарии

| Действие | Где | Ожидание |
|----------|-----|----------|
| Редактирование полей | Карточка заявки (под данными) | Для Админа/Оператора: форма с полями Источник, Адрес, Статус, Подразделение, Тип проблемы, Ответственный (для админа), Описание, Комментарии. Кнопка «Сохранить» |
| Изменить источник/адрес и сохранить | Карточка заявки | После сохранения данные в карточке обновляются |
| Вкладка «Комментарии» | Карточка заявки → вкладка «Комментарии» | Список комментариев (автор, дата, текст). Если нет — «Нет комментариев» |
| Добавить комментарий | Вкладка «Комментарии» → поле «Новый комментарий» → «Добавить комментарий» | Комментарий появляется в списке (для Админ/Оператор) |
| Вкладка «Журнал изменений» | Карточка заявки → вкладка «Журнал изменений» | Список записей: дата, пользователь, действие (CREATE_NOTIFICATION, UPDATE_NOTIFICATION, ADD_COMMENT и т.д.) |

---

## 4. Разграничение прав и роль «Наблюдатель»

| Действие | Где | Ожидание |
|----------|-----|----------|
| Войти как Наблюдатель | Создать пользователя с ролью «Наблюдатель» в Пользователи (под админом) или использовать существующего | В сайдбаре отображается «Наблюдатель (только просмотр)» |
| Меню для Наблюдателя | Тот же вход | В меню **нет** пунктов «Отправить заявку» и «Опросники» |
| Список заявок | Уведомления | Заявки видны, кнопки «Создать уведомление» нет |
| Карточка заявки | Открыть любую заявку | Нет формы редактирования и кнопки «Отправить администратору» |
| Вкладки Комментарии / Журнал изменений | Карточка заявки | Комментарии и журнал **видны**, кнопки «Добавить комментарий» нет (только просмотр) |

---

## 5. Создание заявки с типом проблемы и автором

| Действие | Где | Ожидание |
|----------|-----|----------|
| Создать уведомление | Уведомления → «Создать уведомление» | В форме есть поле «Тип проблемы» (выпадающий список) |
| Заполнить и создать | Источник, Адрес, при желании подразделение и тип проблемы | Заявка создаётся; в списке у неё отображаются «Автор» (текущий пользователь) и выбранный «Тип проблемы» |

## 6. E-mail (IMAP) — опционально

Если настроен IMAP в `backend/.env` (IMAP_HOST, IMAP_USER, IMAP_PASSWORD и т.д.):

```cmd
cd backend
npm run email:ingest
```

Ожидание: в консоли сообщение о количестве обработанных писем и созданных заявок; в системе появляются заявки с источником «Email», адрес = тема письма, описание = отправитель и текст.

Без настройки IMAP скрипт просто выводит, что IMAP не настроен, и завершается без ошибки.

## Быстрая проверка (минимум)

1. Запустить приложение (**Запустить всё.bat**).
2. Войти как **admin**.
3. Уведомления → убедиться, что есть фильтры: подразделение, автор, тип проблемы, местонахождение; в таблице — колонки «Автор», «Тип проблемы», «Ответственный».
4. Открыть любую заявку → проверить редактирование (источник, адрес, статус, тип проблемы, ответственный), вкладки «Комментарии» и «Журнал изменений», добавление комментария.
5. Создать пользователя с ролью «Наблюдатель», выйти, войти под ним → убедиться, что нет «Отправить заявку», «Опросники», нет формы редактирования и кнопки «Отправить администратору», при этом список заявок и просмотр карточки/комментариев/журнала работают.
