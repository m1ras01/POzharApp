# Запуск и тестирование FireNotify

Выполняйте команды **в обычном терминале** (PowerShell или cmd), не в песочнице Cursor.

---

## 1. База данных (PostgreSQL)

Нужна установленная PostgreSQL.

- Создайте базу: `createdb firenotify`  
  или в pgAdmin / любой клиент: создать БД с именем `firenotify`.

- Если логин/пароль не `postgres/postgres`, отредактируйте файл  
  **`backend/.env`** — строка `DATABASE_URL`:
  ```
  DATABASE_URL="postgresql://ВАШ_ЛОГИН:ВАШ_ПАРОЛЬ@localhost:5432/firenotify"
  ```

---

## 2. Backend (API)

Откройте **первый** терминал:

```bash
cd c:\Users\Алекс\Desktop\POzharApp\backend
npm install
npx prisma generate
npx prisma db push
npx prisma db seed
npm run dev
```

Должно появиться: **FireNotify API running at http://localhost:3001**

- Проверка: откройте в браузере http://localhost:3001/api/health  
  Должен быть ответ: `{"ok":true,"service":"FireNotify API"}`

---

## 3. Frontend (интерфейс)

Откройте **второй** терминал (backend не закрывайте):

```bash
cd c:\Users\Алекс\Desktop\POzharApp\frontend
npm install
npm run dev
```

Должно появиться что-то вроде: **Local: http://localhost:5173/**

- Откройте в браузере: **http://localhost:5173**

---

## 4. Тестовый вход

- **Логин и пароль:** заданы в `.env` (ADMIN_LOGIN, ADMIN_PASSWORD) или при первом запуске `npx prisma db seed` в консоль выводится сгенерированный пароль (сохраните его).

После входа проверьте:

- **Панель** — счётчики и последние события  
- **Уведомления** — список (сначала пустой), можно добавить через API или позже через UI  
- **Отправить сообщение** — выбор администратора и отправка  
- **Пользователи** (только для admin) — список, создание/редактирование  
- **Отчёты** — выбор периода и формирование  
- **Настройки** — каналы уведомлений  
- **Центр уведомлений** — статус каналов и список уведомлений  

---

## 5. Если нет PostgreSQL

1. Установите PostgreSQL: https://www.postgresql.org/download/windows/  
2. Либо используйте Docker:  
   `docker run -d -p 5432:5432 -e POSTGRES_PASSWORD=postgres -e POSTGRES_DB=firenotify postgres:15`

После этого повторите шаги 1–2 (создание БД и запуск backend).
