# FireNotify: деплой с нуля — сервер, база, сайт, Telegram

Один сервер = **сайт** + **API** + **база SQLite** + **Telegram-бот**. Всё работает вместе, отдельно базу или «сервер для бота» покупать не нужно.

---

## 1. Что купить (сервер)

Нужен **VPS** (виртуальный сервер) с Linux. Node.js-приложения на обычном хостинге (типа «сайт на PHP») не запускаются — нужен именно VPS.

### Варианты (ориентировочно)

| Провайдер | От | Что даёт |
|-----------|-----|----------|
| **Timeweb Cloud** (РФ) | ~200–300 ₽/мес | 1 CPU, 1 GB RAM — хватит для FireNotify |
| **Selectel** (РФ) | ~300 ₽/мес | Удобная панель, есть тарифы под маленькие проекты |
| **Reg.ru** (РФ) | от ~200 ₽/мес | VPS на выбор |
| **DigitalOcean** | $4–6/мес | Зарубежный, оплата картой |
| **Hetzner** | ~4 €/мес | Недорого в Европе |

**Минимум для FireNotify:** 1 ядро CPU, 512 MB–1 GB RAM, 5–10 GB диск. SQLite не требует отдельного сервера БД — база лежит файлом на том же VPS.

**Домен** — по желанию. Можно сначала зайти по IP: `http://IP_СЕРВЕРА:3000`. Домен потом привяжете в панели и в Nginx.

---

## 2. Что будет на сервере (одним списком)

- **База данных** — один файл `backend/prisma/dev.db` (SQLite). Отдельный «сервер БД» не нужен.
- **Сайт** — то, что видит пользователь в браузере (React), отдаётся тем же Node.js.
- **API** — бэкенд (Express) на том же процессе.
- **Telegram-бот** — запускается вместе с бэкендом, если в `.env` указан `TELEGRAM_BOT_TOKEN`.

Всё это один процесс: `node backend/dist/index.js`.

---

## 3. Подключение к серверу

После оплаты VPS вы получите:
- **IP-адрес** (например `123.45.67.89`)
- **Логин** (часто `root`) и **пароль** или **SSH-ключ**

Подключение с вашего ПК (Windows — через PowerShell или «Терминал» в VS Code; можно PuTTY):

```bash
ssh root@123.45.67.89
```

(подставьте свой IP). Введите пароль при запросе.

## 4. Установка Node.js на сервере

На сервере (после `ssh`):

**Ubuntu/Debian:**

```bash
apt update
apt install -y curl
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs
node -v
```

Должна появиться версия v20.x.x.

**CentOS/AlmaLinux:**

```bash
curl -fsSL https://rpm.nodesource.com/setup_20.x | bash -
yum install -y nodejs
node -v
```

## 5. Как попадает проект на сервер (репозиторий или архив)

### Вариант А: У вас проект в Git (GitHub / GitLab / свой сервер)

На сервере:

```bash
apt install -y git
cd /opt
git clone https://github.com/ВАШ_ЛОГИН/POzharApp.git firenotify
cd firenotify
```

Если репозиторий приватный, настроить SSH-ключ или токен для `git clone`.

### Вариант Б: Залить архивом (без Git)

1. На своём ПК в корне проекта выполните сборку:

```bash
npm run build:deploy
```

2. Создайте архив папок, которые нужны на сервере:
   - `backend/dist`
   - `backend/public`
   - `backend/prisma`
   - `backend/package.json`
   - `backend/package-lock.json`
   (и при желании `backend/src` для правок на сервере)

3. Залить архив на сервер (FileZilla, WinSCP, или `scp`):

```bash
scp firenotify-backend.zip root@123.45.67.89:/opt/
```

4. На сервере:

```bash
cd /opt
unzip firenotify-backend.zip
mv backend firenotify
cd firenotify
```

В итоге на сервере должна быть папка с содержимым **backend**: там же будут `dist`, `public`, `prisma`, `package.json`.

## 7. Файл .env на сервере

В папке **backend** (на сервере это может быть `/opt/firenotify` или `/opt/firenotify/backend` — смотря как вы положили файлы):

```bash
nano .env
```

Создайте файл с содержимым:

```env
PORT=3000
JWT_SECRET=любая_длинная_случайная_строка_без_пробелов
ADMIN_LOGIN=ваш_логин_админа
ADMIN_PASSWORD=надёжный_пароль_не_короче_10_символов
TELEGRAM_BOT_TOKEN=123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

- **PORT** — порт приложения (3000 или 80, если не будете ставить Nginx).
- **JWT_SECRET** — придумайте длинный секрет (логины/сессии).
- **ADMIN_LOGIN** и **ADMIN_PASSWORD** — логин и пароль первого администратора. Задайте сложный пароль (не короче 10 символов). Если ADMIN_PASSWORD не указать — при `npx prisma db seed` будет сгенерирован случайный пароль и выведен в консоль один раз (обязательно сохраните).
- **TELEGRAM_BOT_TOKEN** — токен от @BotFather. Без него сайт и база работают, бот — нет.

Сохранить: `Ctrl+O`, Enter, `Ctrl+X`.

## 8. База данных на сервере (один раз)

База — SQLite, один файл. Создаётся командами в папке backend:

```bash
cd /opt/firenotify
# или cd /opt/firenotify/backend — если backend в подпапке

npx prisma generate
npx prisma db push
npx prisma db seed
```

После этого:
- Появится файл `prisma/dev.db` — это и есть **база сервера**.
- Будет пользователь: логин `admin`, пароль `admin123` — смените после первого входа.

Папку `prisma` и файл `dev.db` при обновлениях **не удаляйте** и делайте бэкапы.

## 9. Запуск (проверка)

```bash
cd /opt/firenotify
node dist/index.js
```

В браузере откройте: `http://IP_СЕРВЕРА:3000`. Должен открыться сайт. Войдите логином и паролем, которые задали в `.env` (ADMIN_LOGIN, ADMIN_PASSWORD) или тем, что был выведен в консоль при seed.

Остановка: `Ctrl+C`.

## 10. Запуск в фоне (PM2), чтобы работало всегда

```bash
npm install -g pm2
cd /opt/firenotify
pm2 start dist/index.js --name firenotify
pm2 save
pm2 startup
```

Команду `pm2 startup` выполните до конца (она выведет команду — её нужно выполнить). После перезагрузки сервера FireNotify поднимется сам.

Полезные команды:

- `pm2 status` — статус
- `pm2 logs firenotify` — логи
- `pm2 restart firenotify` — перезапуск после обновления кода

## 11. Адрес для других: без домена и с доменом

### Если домен не покупаете

**Адрес сайта для других будет таким:**

```
http://IP_ВАШЕГО_СЕРВЕРА:3000
```

Пример: если IP сервера `185.20.45.100`, то адрес: **http://185.20.45.100:3000**

- Этот адрес можно отправлять коллегам — они откроют сайт в браузере.
- Порт **3000** — тот, что указан в `.env` (PORT). Если поставите PORT=80 и запустите от root, адрес будет **http://185.20.45.100** (без :3000).
- Минус: адрес с цифрами и портом не так удобно запомнить и выглядит менее «официально». Для внутреннего использования этого достаточно.

---

### Если покупаете домен — как добавить

1. **Купить домен** (у любого регистратора: Reg.ru, Timeweb, Beget, Cloudflare и т.п.).
2. **Направить домен на сервер:** в панели управления доменом создать **A-запись**:
   - **Имя/хост:** оставить пустым или указать поддомен (например `firenotify` для `firenotify.ваш-домен.kz`).
   - **Значение/IP:** IP вашего VPS (тот же, что в `ssh root@IP`).
   - **TTL:** 300 или 3600.
3. **Подождать 5–30 минут**, пока запись обновится.
4. **На сервере поставить Nginx** и привязать домен к приложению (см. ниже). Тогда сайт будет открываться по адресу вида **http://firenotify.ваш-домен.kz** (или **http://ваш-домен.kz**).

## 12. Домен и Nginx (настройка на сервере)

Чтобы заходить по адресу типа `http://firenotify.ваш-домен.kz` (без порта в адресе):

1. В панели домена настройте A-запись на IP вашего VPS.
2. На сервере установите Nginx:

```bash
apt install -y nginx
```

3. Создайте конфиг:

```bash
nano /etc/nginx/sites-available/firenotify
```

Вставьте (замените `firenotify.ваш-домен.kz` на ваш домен или поддомен):

```nginx
server {
    listen 80;
    server_name firenotify.ваш-домен.kz;
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

Включите сайт и перезапустите Nginx:

```bash
ln -s /etc/nginx/sites-available/firenotify /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

Тогда сайт будет открываться по `http://firenotify.ваш-домен.kz`.

**HTTPS (защищённое соединение):** установите Certbot и получите бесплатный сертификат Let's Encrypt:

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d ваш-домен.kz -d www.ваш-домен.kz
```

Укажите email, согласитесь с условиями. Certbot сам настроит Nginx на HTTPS и при желании перенаправление с HTTP на HTTPS. После этого сайт откроется по `https://ваш-домен.kz`, браузер покажет «Защищено». Сертификат обновляется автоматически.

**Кратко:** A-запись в панели домена → IP сервера. На сервере: Nginx + конфиг с `server_name` вашего домена и `proxy_pass http://127.0.0.1:3000`.

## 13. Telegram

- Бот работает **на том же сервере**, в том же процессе, что и сайт.
- Ничего отдельно «для Telegram» покупать не нужно.
- Достаточно указать **TELEGRAM_BOT_TOKEN** в `.env` и перезапустить приложение (`pm2 restart firenotify`).
- Пользователи в настройках сайта запрашивают код привязки и отправляют боту `/start КОД` — после этого уведомления и заявки приходят в Telegram.

## 15. Обновление проекта на сервере

1. Если через Git: `cd /opt/firenotify && git pull`, затем снова `npm run build:deploy`, `cd backend && npm install --production`.
2. Если архивом: залить новые `dist` и `public`, перезапустить: `pm2 restart firenotify`.
3. Если меняли схему Prisma: на сервере в backend выполнить `npx prisma db push`. Файл `dev.db` не удалять.

Если что-то пойдёт не так — пришлите вывод команд и сообщения об ошибках (без паролей и токенов), подскажу по шагам.
