# Перед пушем в Git и перед деплоем на сервер

## Можно ли пушить в Git (GitHub, GitLab и т.д.)?

**Да.** Проверено:

- В `.gitignore` добавлены:
  - `backend/.env` и `.env` — секреты (пароли, токены) **не попадут** в репозиторий
  - `*.db`, `backend/prisma/*.db` — файлы базы **не попадут**
  - `node_modules/`, `backend/node_modules/`, `frontend/node_modules/`
  - `backend/uploads/`, `backend/public/`, `backend/dist/`, `frontend/dist/`
- Исходный код, батники, документация — всё это можно смело коммитить и пушить.

**Перед первым пушем проверьте:** в репозитории нет файла `backend/.env` (команда `git status` не должен показывать его). Если он уже был добавлен раньше — удалите его из индекса и закоммитьте удаление:  
`git rm --cached backend/.env` затем `git commit -m "Remove .env from repo"`.

## Деплой на сервер

Инструкция: **DEPLOY.md** (кратко) и **ДЕПЛОЙ_С_НУЛЯ.md** (подробно с нуля).

### Кратко

1. **Сборка для деплоя** (на ПК, из корня проекта):
   ```bash
   npm run build:deploy
   ```
   Соберёт фронт, скопирует в `backend/public`, соберёт бэкенд в `backend/dist`.

2. **На сервер** залить:
   - папки `backend/dist`, `backend/public`
   - `backend/package.json`, `backend/package-lock.json`
   - папку `backend/prisma/` (схема, миграции, seed)
   - при первом развёртывании — зависимости: в папке `backend` выполнить `npm install --production`

3. **На сервере создать свой `.env`** в папке `backend` (не копировать с ПК — придумать новые секреты):
   - `PORT=3000`
   - `JWT_SECRET=длинная_случайная_строка`
   - `ADMIN_LOGIN=admin`
   - `ADMIN_PASSWORD=надёжный_пароль_не_короче_10_символов`
   - `TELEGRAM_BOT_TOKEN=токен_от_BotFather` (если нужен бот)

4. **Один раз** в папке `backend` на сервере:
   ```bash
   npx prisma generate
   npx prisma db push
   npx prisma db seed
   ```

5. **Запуск:** `node dist/index.js` (или через PM2/systemd, см. DEPLOY.md).

## Итог

| Действие              | Можно? | Важно |
|-----------------------|--------|--------|
| Пуш в Git             | Да     | Не коммитить `backend/.env` и `*.db` (уже в .gitignore) |
| Деплой на сервер      | Да     | Создать новый `.env` на сервере, не копировать с ПК с реальными паролями/токенами |
