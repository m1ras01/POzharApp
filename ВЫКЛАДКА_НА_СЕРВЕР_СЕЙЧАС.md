# Выкладка FireNotify на сервер — по шагам (CMD / PowerShell)

---

## Обновление через Git (как только что-то обновил)

### 1. На вашем ПК (в папке POzharApp)

```cmd
git add -A
git commit -m "обновление"
git push origin main
```

### 2. На сервере (подключиться: ssh ubuntu@ВАШ_IP или ssh root@ВАШ_IP)

```bash
cd /opt/firenotify
git pull
cd backend
npx prisma generate
cd ..
npm run build:deploy
cd backend
npm install --production
npx prisma db push
pm2 restart firenotify
```

Всё. Сайт обновлён. Адрес: **http://ВАШ_IP:3000**

**Первый раз на сервере** (когда папки `/opt/firenotify` ещё нет):
```bash
cd /opt
git clone https://github.com/ВАШ_ЛОГИН/POzharApp.git firenotify
cd firenotify
npm install
npm run build:deploy
cd backend
npm install --production
nano .env
```
В .env вписать: PORT=3000, JWT_SECRET, ADMIN_LOGIN, ADMIN_PASSWORD, TELEGRAM_BOT_TOKEN. Сохранить (Ctrl+O, Enter, Ctrl+X).
```bash
npx prisma generate
npx prisma db push
npx prisma db seed
pm2 start dist/index.js --name firenotify
pm2 save
pm2 startup
```
Дальше при обновлениях — только блок «Обновление через Git» выше.

## Вариант 1. Проект уже на GitHub (подробно)

### На вашем ПК (если что-то меняли и пушили)

Ничего не нужно — на сервере просто сделаете `git pull` и пересоберёте.

### На сервере (подключитесь: `ssh ubuntu@ВАШ_IP` или `ssh root@ВАШ_IP`)

По очереди введите команды (после каждой — Enter):

```bash
cd /opt/firenotify
```
*(Если папки нет — сначала: `cd /opt`, потом `git clone https://github.com/ВАШ_ЛОГИН/POzharApp.git firenotify`, потом `cd firenotify`)*

```bash
git pull
```

```bash
npm run build:deploy
```

```bash
cd backend
```

```bash
npm install --production
```

```bash
npx prisma generate
```

```bash
npx prisma db push
```

*(При самом первом развёртывании ещё: `npx prisma db seed` — создаст админа. И создайте файл `.env` в папке backend — см. ниже.)*

```bash
pm2 restart firenotify
```
*(При первом разе вместо restart: `pm2 start dist/index.js --name firenotify`, затем `pm2 save` и `pm2 startup`)*

Готово. Сайт: **http://ВАШ_IP:3000**

## Вариант 2. Залить архивом (без GitHub)

### Шаг 1. На вашем ПК (в папке POzharApp)

Откройте **CMD** или **PowerShell** в корне проекта (где папки `backend` и `frontend`):

```cmd
npm run build:deploy
```

Дождитесь окончания. Потом зайдите в папку **backend** и положите в один архив (ZIP):
- папку **dist**
- папку **public**
- папку **prisma**
- файлы **package.json** и **package-lock.json**
  
Назовите архив, например **firenotify.zip**.

### Шаг 2. Залить архив на сервер

- Через **WinSCP** (или аналог): подключитесь к серверу по SFTP, залейте **firenotify.zip** в `/opt`.
- Или из PowerShell на ПК (если установлен OpenSSH/scp):
  ```powershell
  scp C:\Users\Алекс\Desktop\POzharApp\backend\firenotify.zip ubuntu@ВАШ_IP:/opt/
  ```
  *(подставьте свой путь к zip и свой IP)*

### Шаг 3. На сервере (SSH: `ssh ubuntu@ВАШ_IP`)

```bash
cd /opt
```

```bash
unzip -o firenotify.zip -d firenotify
```

```bash
cd firenotify
```
*(если после unzip папка называется иначе — зайдите в неё, там должны быть dist, public, prisma, package.json)*

```bash
npm install --production
```

Создайте файл **.env** (команда `nano .env`), вставьте:

```
PORT=3000
JWT_SECRET=придумайте_длинную_случайную_строку
ADMIN_LOGIN=admin
ADMIN_PASSWORD=ваш_пароль_не_короче_10_символов
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
```

Сохраните: **Ctrl+O**, Enter, **Ctrl+X**.

Дальше:

```bash
npx prisma generate
```

```bash
npx prisma db push
```

```bash
npx prisma db seed
```

Запуск в фоне:

```bash
pm2 start dist/index.js --name firenotify
```

```bash
pm2 save
```

```bash
pm2 startup
```

Скопируйте и выполните команду, которую выведет `pm2 startup`. После этого сайт будет доступен по **http://ВАШ_IP:3000**.

## Кратко

| Где      | Действие |
|----------|----------|
| ПК       | `npm run build:deploy` (и при варианте 2 — собрать ZIP из backend) |
| Сервер   | Загрузить код (git pull или распаковать zip) |
| Сервер   | В папке backend: `npm install --production`, создать `.env` |
| Сервер   | `npx prisma generate`, `npx prisma db push`, при первом разе — `npx prisma db seed` |
| Сервер   | `pm2 start dist/index.js --name firenotify` (или `pm2 restart firenotify` при обновлении) |

Подробные шаги и варианты — в **ДЕПЛОЙ_ПО_ШАГАМ.md** и **DEPLOY.md**.
