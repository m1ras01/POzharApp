# Техническое задание: FireNotify System

## 1. Общая информация

- **Название:** FireNotify System  
- **Назначение:** приём, обработка и передача пожарных уведомлений администраторам, отслеживание статусов и действий сотрудников.

## 2. Цель проекта

Создание программной системы для приёма, обработки и передачи пожарных уведомлений администраторам, отслеживания статуса сообщений и действий сотрудников.

## 3. Основные задачи

- Приём пожарных уведомлений  
- Отображение уведомлений в реальном времени  
- Отправка сообщений администраторам  
- Управление пользователями  
- Хранение истории событий  
- Формирование отчётов  

## 4. Типы пользователей

### 4.1 Администратор
- Просмотр всех уведомлений  
- Получение сообщений  
- Назначение ответственных  
- Управление пользователями  

### 4.2 Оператор
- Приём уведомлений  
- Проверка данных  
- Отправка сообщений администратору  

### 4.3 Наблюдатель (опционально)
- Только просмотр уведомлений  

## 5. Страницы системы

### 5.0 Центр уведомлений
- Подключение Telegram  
- Проверка статуса подключения  
- Список уведомлений  
- Настройки каналов (Push, SMS, Email)  
- История доставленных уведомлений  

### 5.1 Вход
- Логин, пароль, кнопка «Войти»  

### 5.2 Dashboard
- Количество активных/обработанных уведомлений  
- Последние события  
- Быстрые кнопки действий  

### 5.3 Уведомления
- Список, фильтр по дате/статусу, поиск  

### 5.4 Детали уведомления
- Дата/время, источник, адрес, статус, комментарии  
- Кнопка «Отправить администратору»  

### 5.5 Отправка сообщения
- Получатель (администратор), текст, вложения, «Отправить»  

### 5.6 Пользователи
- Список, создание, редактирование, удаление  

### 5.7 Отчёты
- Выбор периода, генерация, экспорт PDF/Excel  

### 5.8 Настройки
- Уведомления, роли, общие параметры  

## 6. Функциональные требования

### 6.0 Уведомления
- Push, SMS, Email, Telegram  
- Привязка Telegram, подтверждение кодом  
- Вкл/выкл типов в настройках  
- Локальные уведомления в браузере  

### 6.1 Уведомления
- Создание, статусы: Новое, В обработке, Проверено, Ложное, Закрыто  

### 6.2 Сообщения
- Отправка администратору, уведомление о доставке, история  

### 6.3 Пользователи
- Регистрация, авторизация, роли и права  

### 6.4 Логи
- Запись действий, просмотр истории  

## 7–9. Нефункциональные и безопасность

- Веб-приложение, Chrome/Edge/Firefox, отклик до 2 с  
- Резервное копирование, защита паролей  
- HTTPS, хеширование паролей, разграничение доступа  

## 10. Этапы

Анализ → Проектирование → Разработка → Тестирование → Внедрение → Поддержка  

## 11. Результат

Рабочая система с удобным интерфейсом и надёжным хранением данных.
