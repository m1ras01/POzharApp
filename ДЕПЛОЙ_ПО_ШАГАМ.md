# FireNotify: деплой по шагам (для новичков)

Пошаговая инструкция: что нажимать и что вводить. Делайте строго по порядку.

---

## Часть 1. Подключиться к серверу

### Шаг 1.1. Узнать публичный IP и пароль

1. Откройте сайт **PS Cloud** и войдите в личный кабинет.
2. Найдите раздел **«Серверы»** или **«Виртуальные машины»** / **«VPS»**.
3. Нажмите на **ваш сервер** (тот, что вы купили).
4. На странице сервера найдите блок **«Сеть»** или **«IP»**.
5. Найдите **внешний (публичный) IP** — это адрес вида `185.xxx.xxx.xxx` или `5.xxx.xxx.xxx` (не 10.202.x.x).
6. Запомните или запишите:
   - **Публичный IP** (например `185.20.45.100`)
   - **Пароль** от сервера (его вы задавали при создании или он был в письме/в панели).

Если публичного IP нет в списке — на той же странице может быть кнопка **«Консоль»** или **«VNC»**. Нажмите её — откроется чёрное окно (терминал в браузере). Переходите к **Части 2** и делайте всё из консоли в браузере (тогда SSH с ПК не нужен).

---

### Шаг 1.2. Открыть терминал на вашем ПК

1. Нажмите на клавиатуре **Win + R** (клавиша Windows и буква R).
2. В окошке введите: `powershell` и нажмите **Enter**.
3. Откроется синее или чёрное окно с текстом вроде `PS C:\Users\...>` — это **терминал** (PowerShell).

---

### Шаг 1.3. Подключиться к серверу по SSH

1. В письме/панели может быть указан пользователь **ubuntu**, а не root. Тогда подключайтесь так:

```
ssh ubuntu@185.20.45.100
```

Если указан **root** — используйте `ssh root@185.20.45.100`. Подставьте **свой** IP вместо `185.20.45.100`.

2. Нажмите **Enter**.
3. Если спросят «Are you sure you want to continue connecting?» — введите `yes` и нажмите **Enter**.
4. Когда попросят пароль — **вставьте пароль от сервера** (правой кнопкой мыши). Символы при вводе не видны — это нормально. Нажмите **Enter**.

Если всё верно, в начале строки появится что-то вроде `ubuntu@сервер:~$` или `root@сервер:~#` — значит вы **внутри сервера**.

**Если вошли как ubuntu** и нужны права администратора — введите `sudo -i`, нажмите Enter, при запросе введите тот же пароль. После этого будете под root (`root@сервер:~#`).

Если пишет «Connection refused» — вы использовали внутренний IP (10.x.x.x). Вернитесь к шагу 1.1 и найдите **публичный** IP.

---

## Часть 2. Установить Node.js на сервере

Все команды ниже вводите **в том же окне** (где вы уже подключены к серверу). После каждой команды нажимайте **Enter** и ждите окончания (иногда 1–2 минуты).

**Команда 1:**

```
apt update
```

**Команда 2:**

```
apt install -y curl
```

**Команда 3:**

```
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
```

**Команда 4:**

```
apt install -y nodejs
```

**Команда 5 (проверка):**

```
node -v
```

Должно вывести что-то вроде `v20.10.0`. Если видите версию — Node.js установлен. Идём дальше.

## Часть 3. Загрузить проект на сервер

Два варианта: **архив через WinSCP** (проще) или **через GitHub** (удобнее обновлять). Выберите один.

---

### Вариант А: через GitHub (рекомендуется для обновлений)

#### Шаг А.1. Выложить проект на GitHub (один раз, на вашем ПК)

1. Зарегистрируйтесь на **https://github.com** (если ещё нет аккаунта).
2. Нажмите **«+»** → **«New repository»**. Название, например **POzharApp** или **FireNotify**. Репозиторий можно сделать **Private**. Нажмите **«Create repository»**.
3. На странице нового репозитория скопируйте **URL** (например `https://github.com/ВАШ_ЛОГИН/POzharApp.git`).
4. На вашем ПК откройте папку **POzharApp** (корень проекта, где лежат папки `backend` и `frontend`) и откройте там терминал (PowerShell). **Важно:** загружать через кнопку «Upload files» на GitHub нельзя — папки не сохраняются. Нужно отправить проект через git (команды ниже).
5. Выполните по очереди (подставьте свой URL):

```bash
git init
git add .
git commit -m "FireNotify initial"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/POzharApp.git
git push -u origin main
```

При первом `git push` GitHub попросит войти (логин и пароль или токен). Если репозиторий приватный — используйте **Personal Access Token** вместо пароля (GitHub → Settings → Developer settings → Personal access tokens).

После успешного push проект будет на GitHub.

#### Шаг А.2. На сервере: клонировать и собрать

Подключитесь по SSH: `ssh ubuntu@46.247.42.72` (подставьте свой IP).

**1. Установить git:**
```bash
sudo apt install -y git
```

**2. Клонировать репозиторий** (подставьте свой URL — тот же, что на GitHub):
```bash
cd /opt
git clone https://github.com/ВАШ_ЛОГИН/POzharApp.git firenotify
cd firenotify
```

Если репозиторий **приватный**, при `git clone` попросит логин и пароль (или токен). Введите их.

**3. Установить зависимости и собрать проект:**
```bash
npm install
npm run build:deploy
cd backend
npm install --production
```

Дальше переходите к **Части 4** (создание файла .env на сервере).

**Обновление проекта позже:** на ПК делаете изменения, затем `git add .`, `git commit -m "описание"`, `git push`. На сервере: `cd /opt/firenotify`, `git pull`, `npm run build:deploy`, `cd backend && npm install --production`, `pm2 restart firenotify`.

---

### Вариант Б: залить архив с вашего ПК (без GitHub)

### Шаг 3.1. На вашем ПК (в папке проекта)

1. Откройте папку проекта **POzharApp** (где лежат папки `frontend`, `backend`).
2. Зажмите **Shift**, нажмите **правой кнопкой** по пустому месту в папке → выберите **«Открыть окно PowerShell здесь»** или **«Открыть в терминале»**.
3. Введите и нажмите Enter:

```
npm run build:deploy
```

Подождите, пока сборка закончится (1–3 минуты).

4. Создайте архив:
   - Зайдите в папку **backend**.
   - Выделите папки и файлы: **dist**, **public**, **prisma**, **package.json**, **package-lock.json** (можно ещё **src**).
   - Правой кнопкой → **«Отправить»** → **«Сжатая ZIP-папка»**. Назовите архив, например **firenotify.zip**.

### Шаг 3.2. Залить архив на сервер (через программу)

1. Скачайте **WinSCP**: https://winscp.net/eng/download.php (бесплатно).
2. Установите и откройте WinSCP.
3. В окне входа:
   - **Протокол:** SCP (или SFTP).
   - **Хост:** ваш публичный IP (например `185.20.45.100`).
   - **Порт:** 22.
   - **Имя пользователя:** root.
   - **Пароль:** пароль от сервера.
4. Нажмите **«Вход»**. Слева — ваш ПК, справа — сервер.
5. Справа перейдите в папку **/opt** (дважды клик по ней).
6. Перетащите ваш **firenotify.zip** с левой части в правую (в /opt).
7. Дождитесь окончания загрузки.

### Шаг 3.3. Распаковать архив на сервере

Вернитесь в окно **PowerShell**, где вы подключены к серверу по SSH.

**Команда 1 (установить unzip, если его нет):**

```
apt install -y unzip
```

**Команда 2 (перейти в /opt):**

```
cd /opt
```

**Команда 3 (распаковать архив — имя файла может быть другим):**

```
unzip firenotify.zip
```

**Команда 4 (посмотреть, что получилось):**

```
ls
```

Должны быть папки **dist**, **public**, **prisma** и файлы **package.json**, **package-lock.json**. Если они лежат внутри папки (например внутри **backend**), перейдите в неё:

```
cd backend
```

(или как называется папка после распаковки). В итоге вы должны быть в папке, где лежат **dist** и **package.json**.

**Команда 5 (установить зависимости):**

```
npm install --production
```

Подождите 1–2 минуты.

## Часть 4. Создать файл .env на сервере

В той же SSH-сессии (в папке, где лежит package.json).

**Команда 1 (открыть редактор):**

```
nano .env
```

Откроется простой редактор.

**Вставьте эти строки** (замените на свои данные):

```
PORT=3000
JWT_SECRET=придумайте_длинную_случайную_строку_без_пробелов
ADMIN_LOGIN=admin
ADMIN_PASSWORD=ваш_надёжный_пароль_минимум_10_символов
TELEGRAM_BOT_TOKEN=
```

- **ADMIN_LOGIN** — логин, под которым будете входить на сайт.
- **ADMIN_PASSWORD** — пароль (не короче 10 символов, буквы+цифры лучше).
- **TELEGRAM_BOT_TOKEN** — если есть токен от @BotFather, вставьте после `=`. Если нет — оставьте пустым.

**Сохранить и выйти:**
- Нажмите **Ctrl+O**, затем **Enter** (сохранить).
- Нажмите **Ctrl+X** (выйти).

## Часть 5. Создать базу данных и администратора

В той же папке на сервере выполните по очереди:

```
npx prisma generate
```

```
npx prisma db push
```

```
npx prisma db seed
```

После последней команды должно написать что-то вроде «Seed: администратор создан». Если в .env не был задан ADMIN_PASSWORD, в консоль выведется случайный пароль — **скопируйте и сохраните его**, им нужно будет войти на сайт.

---

## Часть 6. Запустить сайт

**Проверка (вручную):**

```
node dist/index.js
```

Не должно быть красных ошибок. В браузере на вашем ПК откройте: **http://ВАШ_ПУБЛИЧНЫЙ_IP:3000** (подставьте свой IP). Должна открыться страница входа FireNotify. Логин и пароль — те, что в .env (или сгенерированный при seed).

Остановить сервер: нажмите **Ctrl+C**.

**Запуск навсегда (чтобы работало после закрытия окна):**

```
npm install -g pm2
```

```
pm2 start dist/index.js --name firenotify
```

```
pm2 save
```

```
pm2 startup
```

Последняя команда выведет строку вида `sudo env PATH=...`. **Скопируйте и выполните эту строку** (вставьте в терминал и нажмите Enter). После этого при перезагрузке сервера FireNotify будет запускаться сам.

---

## Часть 7. Открыть порт 3000 (если сайт не открывается)

Если по адресу http://IP:3000 браузер пишет «Не удаётся подключиться»:

На сервере выполните:

```
apt install -y ufw
ufw allow 22
ufw allow 3000
ufw --force enable
```

Также в панели **PS Cloud** найдите настройки **файрвола** или **«Группа безопасности»** для вашего сервера и добавьте правило: **входящий трафик, порт 3000, разрешить**.

## Краткая шпаргалка

| Что делаем | Где | Команда/действие |
|------------|-----|-------------------|
| Подключиться к серверу | ПК, PowerShell | `ssh root@ПУБЛИЧНЫЙ_IP` |
| Установить Node.js | Сервер | `apt update` → `apt install -y curl` → команды из Части 2 |
| Залить проект | ПК: сборка + архив; WinSCP: залить в /opt | см. Часть 3 |
| Распаковать и установить зависимости | Сервер, /opt | `unzip firenotify.zip`, `cd` в папку, `npm install --production` |
| Создать .env | Сервер | `nano .env` — вставить PORT, JWT_SECRET, ADMIN_*, TELEGRAM_* |
| Создать базу | Сервер | `npx prisma generate`, `db push`, `db seed` |
| Запустить | Сервер | `pm2 start dist/index.js --name firenotify`, `pm2 save`, `pm2 startup` |
| Открыть сайт | Браузер на ПК | http://ПУБЛИЧНЫЙ_IP:3000 |
